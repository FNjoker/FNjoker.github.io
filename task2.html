一、关系型数据库和非关系型数据库

1.什么是关系型数据库

（1）关系型数据库是依据关系模型来创建的数据库，所谓关系模型就是指二维表格模型，就是“一对一、一对多、多对多”等关系。关系模型包括数据结构（数据存储问题，二维表）、操作指令集合（sql语句）、完整性约束（表内数据约束、表与表之间的约束）

（2）常见的关系型数据库：Oracle、DB2、PostgreSQL、Microsoft SQL Server、Microsoft Access、MySQL

（3）关系型数据库的特点：安全、容易理解、不节省空间（即使某些值为空也要分配字段）

2.什么是非关系型数据库

（1）非关系型数据库主要是基于“非关系模型”的数据库，非关系型数据库有列模型（存储的数据是一列列的，以一列为一个记录）、键值对模型（存储的数据是一个个“键值对”）、文档类模型（以一个个文档来存储数据）

（2）常见的非关系型数据库：Hbase、redis、MemcacheDB、mongoDB

（3）非关系型数据库的特点：性能高、非常容易扩展

sql inner join从多个表中返回满足join条件的所有行，取的是table1和table2的交集；sql left join从左表返回所有行，右表如果没有匹配则返回结果为null，相当于取得是table1+table1和table2的交集；sql right join从右表返回所有的行，左表如果没有匹配则返回结果为null，相当于取的是table2+table1和table2的交集

在使用join时，on和where条件的区别如下：
1、on条件是在生成临时表时使用的条件，它不管on中的条件是否为真,都会返回左边表中的记录
2、where条件是在临时表生成好后，再对临时表进行过滤的条件。这时已经没有left join 的含义了，条件不为真的全部过滤掉。

三、MySQL学习

1.MySQL简介

MySQL 是一个关系型数据库管理系统，由瑞典 MySQL AB 公司开发，目前属于 Oracle 公司。MySQL 是一种关联数据库管理系统，关联数据库将数据保存在不同的表中，而不是将所有数据放在一个大仓库内，这样就增加了速度并提高了灵活性。

  • MySQL 是开源的，所以你不需要支付额外的费用。
  • MySQL 支持大型的数据库。可以处理拥有上千万条记录的大型数据库。
  • MySQL 使用标准的 SQL 数据语言形式。
  • MySQL 可以运行于多个系统上，并且支持多种语言。这些编程语言包括 C、C++、Python、Java、Perl、PHP、Eiffel、Ruby 和 Tcl 等。
  • MySQL 对PHP有很好的支持，PHP 是目前最流行的 Web 开发语言。
  • MySQL 支持大型数据库，支持 5000 万条记录的数据仓库，32 位系统表文件最大可支持 4GB，64 位系统支持最大的表文件为8TB。
  • MySQL 是可以定制的，采用了 GPL 协议，你可以修改源码来开发自己的 MySQL 系统。

（10）sql约束用于规定表中的数据规则。如果存在违反约束的数据行为，行为会被约束终止。约束可以在创建表时规定（create table），或在创建表后规定（alter table）。

约束条件有： （1）NOT NULL -指示某列不能存储NULL值
（2）UNIQUE-保证某列的每行必须有唯一的值
（3）PRIMARY KEY-NOT NULL 和 UNIQUE的结合。确保某列有唯一标识
（4）FOREIGN KEY-保证一个表中的数据匹配另一个表中的值参照完整性
（5）CHECK-保证列中的值符合制定的条件
（6）DEFAULT-规定没有给列赋值时的默认值
